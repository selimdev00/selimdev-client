# Deploy

Static Nuxt 3 site deployed to Cloudflare Pages from GitHub.

## Cloudflare Pages settings

When creating the Pages project (Workers & Pages > Create > Pages > Connect to Git):

| Setting | Value |
|---|---|
| Production branch | `master` |
| Build command | `npm run generate` |
| Build output directory | `.output/public` |
| Root directory | `/` |
| Node version | `20` (read from `.nvmrc`) |

### Environment variables

| Name | Production value |
|---|---|
| `NUXT_PUBLIC_SITE_URL` | `https://selim.services` |
| `NODE_VERSION` | `20` (only if `.nvmrc` is ignored) |

### Custom domain

After the first deploy succeeds, attach `selim.services` (and the `www.` redirect if desired) under Pages > Custom domains. DNS lives in the same Cloudflare account, so it auto-validates.

## Local build sanity check

```bash
npm run generate
npx serve .output/public
```

## Notes

- The site is **fully static** (`nuxt generate`). No SSR, no edge functions. `_worker.js` is not generated.
- `public/_headers` carries cache headers for assets and security headers (X-Frame-Options, Referrer-Policy, Permissions-Policy). Cloudflare reads it at the site root.
- `i18n` strategy is `prefix_except_default` - English at `/`, Russian at `/ru/`. Both locales are pre-rendered.
- Sitemap is at `/sitemap.xml`, generated by `@nuxtjs/sitemap` from `runtimeConfig.public.siteUrl`.
