# Deploy

Static Astro 5 site deployed to Cloudflare Pages from GitHub.

## Cloudflare Pages settings

When creating the Pages project (Workers & Pages > Create > Pages > Connect to Git):

| Setting | Value |
|---|---|
| Production branch | `master` |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Root directory | `/` |
| Node version | `20` (read from `.nvmrc`) |

### Environment variables

| Name | Production value |
|---|---|
| `NODE_VERSION` | `20` (only if `.nvmrc` is ignored) |

The Astro `site` URL (`https://selim.services`) is hard-coded in `astro.config.mjs` and `src/lib/site.ts`; no env override needed.

### Custom domain

After the first deploy succeeds, attach `selim.services` (and the `www.` redirect if desired) under Pages > Custom domains. DNS lives in the same Cloudflare account, so it auto-validates.

## Local build sanity check

```bash
npm run build
npx serve dist
```

## Notes

- The site is **fully static** (`astro build`). No SSR, no edge functions, no `_worker.js`.
- `public/_headers` carries cache headers for assets and security headers (X-Frame-Options, Referrer-Policy, Permissions-Policy). Cloudflare reads it at the site root.
- i18n via Astro's built-in routing - English at `/`, Russian at `/ru/`. Both locales are pre-rendered to static HTML.
- Sitemap index is at `/sitemap-index.xml`, generated by `@astrojs/sitemap`.
- Inter font is self-hosted via `@fontsource/inter` (no Google Fonts CDN call).
- Islands (Preact) are loaded as needed: `client:load` for theme toggle and bottom panel, `client:idle` for the cursor gradient, `client:visible` for image viewers. Total client JS is around 30 KB across all islands.
